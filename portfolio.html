Game Description

This is a "cursor" game where the user moves the cursor around the map
to reach the finish line without touching anything yellow. The timer
is set to 120 seconds and decreases by one every second. The amount
of time you finish is the amount of points you get. When the player
reaches the second and third round, the randomness occurs when the
yellow sprites move randomly and tries to tag the user. If the player
touches yellow, they automatically lose. The rounds get harder since
there are more yellow objects to avoid each round. If the user finishes all
the rounds, there is a bonus round at the end. The bonus round consists
of three questions that the user has to answer by choosing the correct
path and not touch yellow at the same time. Each correct answer is
plus 100 points. If the user chooses the wrong path, they lose.

Game Reflection

I was inspired to make a game based on the "if touch color ?" block
after finding exploring the different blocks in Scratch. Two successful
points was that I was able to reuse the same rules of not touching yellow
for different maps and different types of rounds and that I was able
to manage a bunch of different sprites that popped up during different
rounds. Two obstacles that I face was keeping track of the number of
sprites and coming up with different maps. I overcame the first obstacles
by using a lot of "broadcast" blocks and overcame the second obstacles
by searching up different types of mini games online. If I had more
time, I would create more rounds and add better visuals and sounds.
An, important algorithm that contributes significantly to the game
was the "not-touching-yellow" rule. The game was made based on this
rule since the user can only win by not touching yellow.



CYOA: Journey to Pynsthena

My story is about the player and a healer, named Jayden, who goes on
a journey home after guarding a caravan of people. Using their gold,
meal packs, and healing potions, they go on an adventure that depends
on the user to determine their fate.

Amy and I made a program using a flowchart to plan out our story. We
split the program into two parts: I coded everything that happened
if the user chose to kill the wolf (kill_wolf()) while Amy coded everything
that happened if the user tamed the wolf (tame_wolf()). We coded and
linked each corresponding scene to each other and tested the entire
code at the end and looked for errors. Some loops that we had were
the reset loop and the invalid input loop. We used the for loop to
create a loop that allowed the user to reset the program without
having to press run and used the while loop to allow the user to type
a valid answer to continue to the next scene. Two problems that I
had were that the amount of gold, healing potions, and meal packs
didn't add up and that the next scene wasn't playing. I resolved the
first problem by using variables to keep track of the amount and
resolved the second problem by going through all the different
scenes and choices and checking to make sure they led to the correct
scene.
Organizing the scenes into different functions was useful since it
removed duplicates of the same scene. Also, it helps keep track of which
block of code is which scene, making it easier to read.
Two CYOA stories that I liked were "Benny's Adventures" and "Plankton
Strikes Back" since they were really original and we were only able
to have time to read two other stories.
